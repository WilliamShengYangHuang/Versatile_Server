第一步 · 准备一台机器
内部使用为主,不一定要租云主机 —— 你自己的 Windows 电脑或一台 Linux 机器都可以,只要能被团队成员访问到即可(同一局域网,或后续用 VPN/内网穿透)。
选择你的机器类型:
最低要求:1 核 CPU · 1GB 内存 · 有终端(PowerShell / SSH)访问权限。
默认部署为「内部模式」:服务只在这台机器本机监听,团队通过内网/VPN 访问,不需要公网 IP、不需要备案。
第二步 · 一键安装后端 ({{ platformLabel }})
把项目里的 backend/ 文件夹传到你的机器上(scp / U 盘 / git clone 均可),然后在该目录下执行:
cd backend
powershell -ExecutionPolicy Bypass -File install.ps1
请以管理员身份打开 PowerShell 运行。脚本会自动:通过 winget 安装 Node.js(若没有)→ 安装依赖 → 生成管理员 Token → 注册后台自启动(有 NSSM 则注册为 Windows 服务,否则用任务计划程序在每次登录时自动运行)。完成后会打印本机访问地址与 Token。 
cd backend
chmod +x install.sh
sudo bash install.sh
脚本会自动:安装 Node.js(若没有)→ 安装依赖 → 生成管理员 Token → 注册为 systemd 服务(开机自启)→ 启动服务。完成后会打印出本机访问地址与 Token,记下它们,下一步要用到。
默认内部模式,只监听 127.0.0.1:8787。若要保留未来外部访问入口,见第四步。

第四步 · 内部使用 / 预留外部接口
当前推荐保持「内部模式」。如果未来需要让外部用户访问某些接口(例如公开的 AI 模型 API),再按以下方式切换,无需重装:
powershell -ExecutionPolicy Bypass -File install.ps1 -Mode external -Port 8787 -AdminToken 你的Token
外部模式会监听 0.0.0.0,并要求所有写操作(添加/删除)带上 Authorization: Bearer <Token>。需要在 Windows 防火墙里放行对应端口的入站规则,并建议在前面再套一层反向代理 + HTTPS。 
sudo MODE=external PORT=8787 ADMIN_TOKEN=你的Token bash install.sh
外部模式会监听 0.0.0.0,并要求所有写操作(添加/删除)带上 Authorization: Bearer <Token>。建议再套一层 nginx + HTTPS,并只对外开放确需公开的接口(如模型调用 API),其余仍保持内部访问。
提示:先跑内部模式验证一切正常,外部开放是后续按需再做的一步,不影响现在使用。
完成
后端已就绪。回到控制台,右上角「数据源设置」里填入同样的后端地址并保存,即可用真实数据替换空态。

日常运维:systemctl status server-console-backend 查看状态,journalctl -u server-console-backend -f 看日志。
